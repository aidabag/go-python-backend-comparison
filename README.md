# Сравнение производительности Go и Python

Данный проект представляет собой монорепозиторий для высокоточного тестирования и сравнения производительности бэкенд-сервисов, написанных на **Go** и **Python**. Сравнение проводится в идентичных условиях с использованием единой SQL-схемы БД и структуры контейнеризации.

## Запуск проекта (через Docker Compose)

### Старт среды
Запуск базы данных и текущих сервисов. Флаг `--build` принудительно пересобирает бинарники
```bash
docker-compose up -d --build
```

### Честный сброс данных (Для бенчмарков)
**Крайне Важно!** Для чистоты каждого отдельного тестового прогона (например, перед сравнением разных языков) необходимо полностью уничтожать диск базы данных, чтобы она создавалась и заполнялась с абсолютного нуля.
```bash
docker-compose down -v
```

## Тестирование API (cURL запросы)

Ниже представлена полная спецификация CRUD-операций для ручной проверки работы системы с помощью утилиты `curl`.

### Системные запросы
```bash
# Проверить, готов ли сервис принимать запросы
curl http://localhost:8080/health

# Получить внутренние метрики состояния памяти и HTTP (Prometheus)
curl http://localhost:8080/metrics
```

---

### Управление товарами

**1. Создать товар**
```bash
curl -X POST http://localhost:8080/products \
     -d '{"name": "Laptop", "price": 100000, "stock": 50}'
```

**2. Получить список всех товаров**
```bash
curl http://localhost:8080/products
```

**3. Получить конкретный товар по его ID**
```bash
curl http://localhost:8080/products/1
```

**4. Обновить данные товара (Например, поменять цену)**
```bash
curl -X PUT http://localhost:8080/products/1 \
     -d '{"name": "Laptop Pro", "price": 120000, "stock": 45}'
```

**5. Удалить товар**
```bash
curl -X DELETE http://localhost:8080/products/1
```

---

### Управление заказами (Orders & Items)

**1. Создать новый (пустой) заказ**
```bash
curl -X POST http://localhost:8080/orders \
     -d '{"status": "new"}'
```

**2. Добавить товар в заказ**
*(В этот момент в БД автоматически произойдет транзакция и со `stock` спишется 2 единицы товара)*
```bash
curl -X POST http://localhost:8080/orders/1/items \
     -d '{"product_id": 1, "quantity": 2}'
```

**3. Просмотреть список заказов**
```bash
curl http://localhost:8080/orders
```

**4. Получить заказ по его ID**
```bash
curl http://localhost:8080/orders/1
```

**5. Удалить позицию из заказа**
*(Товар автоматически "вернется" на склад, а его `stock` увеличится)*
```bash
curl -X DELETE http://localhost:8080/orders/1/items/1
```

**6. Получить итоговую сумму заказа**
```bash
curl http://localhost:8080/orders/1/total
```

**7. Получить детальную квитанцию по заказу**
```bash
curl http://localhost:8080/orders/1/summary
```

### Аналитика

Аналитические SQL-запросы для оценки производительности выборки.

**1. Запросить среднюю стоимость существующих заказов**
```bash
curl http://localhost:8080/analytics/orders/average
```

**2. Вывести Топ самых продаваемых товаров**
```bash
curl http://localhost:8080/analytics/products/top
```
